
 BLACK CHAT 

 Если кому-то нужен маленький и удобный чат просто для общения без "украшений" и который
  экономит трафик - то этот чат подходит для этого на все 100%.  
  Движок полностью отлажен и "вылизан", внутри скриптов есть подробные комментарии, функции
  сгрупированы по функционалу.

 Клиентская часть выполнена по техлогии XHTML, CSS, JavaScript и имеет одну особенность - в клиенте
  испльзуется менеджер задач, т.е. любые операции сначала помещаются в специальную очередь,
  а потом выполняются, следующая задача выполняется если предыдущая завершилась успешно.
 
 Серверная часть базируется на PHP, используются естественно сессии, данные хранятся в обычных файлах. 
  Все разбито по функциям, в PHP скрипте имеются комментарии ко всем функциям. 
 
 По поводу прав - используются 4 уровня привелегий: 0 - гость, 1 - пользователь, 2 - админ,
  3 - самый главный (по умолчанию им становится самый первый зарегистрированный пользователь в чате,
  можно просто открыть файл профиля и поставить 3 вместо 1, всё хранится в виде обычного текста). Админов
  назначает самый главный через админку. Преимущества имеет только тот у кого права выше,
  т.е. если скажем админ не сможет отправить другому админу скрипт, зато админ может отправить и 
  пользователю и гостю, а главный может отправить всем и вся, на то он и главный, данная политика глобальна
  для всех операций.

 По поводу использования различных символов в именах - недопустимо использовать только следующие 
  символы: '"<>~ остальные использовать можно, по умолчанию можно лишь использовать @1234567890 и пробелы,
  тильда ~ является служебным символом, этот символ резделитель формирует "язык" для взаимодействия
  между сервером и клиентом.

 Функционал чата (- означает что данный параметр можно вырубить через админку): 
  
  +- Защита от флуда (см. подробно главный скрипт) 
  +- Анализатор ненормативной лексики (см. подробно главный скрипт)
  +- Гостевой вход
  +- Регистрация новых пользователей
  +- Запись всех сообщений в лог
  +- Возможность запускать неограниченное количество клиентов под одной сессией
  +- Автоудаление старых пользователей из базы, если после последнего посещения прошёл 1 год.
  +  Возможность бана (если гость - то банится и по логину и по IP, если зарегиный пользователь - 
     банится только по логину). Банить можно на минуты, часы и дни. Банить могут только юзеры с 
     уровнем привелегий > 1, банить можно только тех у кого ниже привелегии. В некоторых случаях
     сервер банит сам, например если юзер флудит несколько раз всподряд.
  +  Возможность редактирования/удаления личного профиля
  +  Привелегии
     0 - Гость
     1 - Пользователь
     2 - Администратор
     3 - Главный
  +  Возможность блокировки зарегистрированного пользователя через админку
  +  Админка, админы могут редактировать чужие профили, удалять пользователей из чата, менять настройки чата,
     просмотривать лог.
  +  Служба воостановления пароля через емеил.
  +  Возможность удалять забаненных юзеров через админку (если в бан-листе кто-то есть, то весь список
     будет выведен в настройках чата, см. скрины).
  +  Возможность отправлять сообщения для кого-то публично или анонимно (см. скрины).
  +  Игнорирование, т.е. сообщения игнорированных пользователей просто будут игнорироватся.
  +  Режим реверса (R) при котором смысл игнора меняется в противоположную сторону, т.е. то что отмечено будет 
     разрешено, а не отмеченные пользователи будут игнорироватся.
  +  Режим привата (P) при котором на клиенте будут отображатся только личные сообщения и сообщения сервера,
     клиент сможет отправлять только лично кому-то сообщения, игнор работает так-же как и в обычном режиме.
  +  Возможность отправки тегов в том числе и скриптовых. Теги могут отправлять только юзеры с уровнем > 1
     и только тем у кого права ниже, в остальных случаях теги будут переданы как обычный текст.
     Скрипты принимаются даже если отправитель будет игнорирован со стороны клиента.
     Наример если отправить это: <script>alert("Пошли все вон!"); top.WF.LogOut();</script>
     ...то все пользователи у которых ниже права полетять из чата, т.е. гости и пользователи, если
     этот скрипт отправил главный то и админы тоже нахрен полетят.

 Преимущества данного чата:
  - Низкий уровень трафика по сравнению с другими чатами
  - Возможность работы на предельно низкой скорости канала 
  - Маленький размер
  - Клиентская часть может работать в любых браузерах начиная с IE 4
  - Серверный скрипт может работать начиная с PHP 4.1   
  - Все функции оптимизированны в отношении компактности и быстродействия, хотя при таких размерах
    скрипта это в общем-то и не нужно, просто старая привычка...
  - Простой и понятный интерфейс.

 Описание клиентской части:
 
  Флажки:
   "F" - Постоянный фокусировка в окне ввода сообщений
   "S" - Автоскроллинг, т.е. автоматическая прокрутка окна сообщений
   "R" - см. выше
   "P" - см. выше
 
  Кнопка [CLI] - очищает игнор лист, за ней находится индикатор текущего состояния канала, 
   Зелёный цвет означает что идёт обмен данными между клиентом и сервером (на нормальных каналах
    даже не загорается),
   Жёлтый и красные цвета означют проблемы со связью.

  Время - время сервера, постоянно синхронизируется.

  Остальное всё просто.

  В случае нарушения свзи с сервером, клиентский скрипт будет до последнего пытатся восстановить связь.

 По поводу админки, пользователи которые зарегистрировались не больше трёх суток назад отображаются
  зелёным цветом, в остальном все просто.

 Скрипт протестирован и оптимизирован на все 100%.

 Описание директорий и файлов:
  
  base - в ней хранится база данных зарегиных пользователей, также бан лист, и файл настроек чата.
  client - скрипты для клиента
  server - все остальные скрипты, стили, html файлы и пр.
  session - в ней хранятся активные профили пользователей, файлы с сообщениями, лог файл с сообщениями.

 Обработкой сообщений, т.е. приведение к надлежащему виду, занимается клиентский скрипт, сервер
  просто "подсказывает" как нужно вывести то или иное сообщение.

 Для нормальной работы серверного скрипта необходимо чтобы скрипт имел полный доступ к папкам 
  "base" и "session" и того что находится внутри них. В скрипте используются супер глобальные
  переменные: $GLOBALS, $_SESSION и $_REQUEST.

 Скрипт способен работать на PHP версии >= 4.3.
   
 Для нормальной работы клиентского скрипта необходимо чтобы в браузере были включены
  куки и поддержка JavaScript, что обычно по умолчанию.
  
 В скриптах используются естественно относительные пути, в любой директории будет работать.

 По поводу безопасности - целиком ложится на сервер и клиента, в самом PHP скрипте
 безопасность реализована полностью.

 По поводу синтаксических проверерок - всё реализовано на высшем уровне;-)
